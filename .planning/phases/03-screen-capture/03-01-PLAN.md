<objective>
Build the native Swift helper library (GaimerScreenCapture) that wraps ScreenCaptureKit's SCScreenshotManager API and exposes C-callable functions via @_cdecl for P/Invoke consumption from .NET.

Purpose: ScreenCaptureKit has no managed C# bindings in .NET 8 MacCatalyst. A native Swift helper is the only reliable way to call SCK from the MAUI app. This plan creates the helper, builds it as an xcframework, and links it into the MAUI project.

Output: A compiled xcframework linked into the MAUI project that exports `sck_capture_window` and `sck_is_available` functions callable via P/Invoke.
</objective>

**Part B: Create the Swift source file**

Create `src/GaimerDesktop/NativeHelpers/GaimerScreenCapture/Sources/GaimerScreenCapture/GaimerScreenCapture.swift`.

This file wraps ScreenCaptureKit and exports C-callable functions using `@_cdecl`. Use the @_cdecl + C function pointer pattern (NOT ObjC block marshaling) because P/Invoke with function pointers is better understood in .NET than ObjC block marshaling.

The file must contain these exported functions:

**1. `sck_is_available() -> Bool`**
- Returns true if ScreenCaptureKit is available on this system
- Check using `if #available(macCatalyst 18.2, macOS 14.0, *)` guard
- This lets C# decide whether to use SCK or fall back to CGDisplayCreateImage

**2. `sck_capture_window(windowID: UInt32, width: Int32, height: Int32, callback: @convention(c) (UnsafePointer<UInt8>?, Int32) -> Void)`**
- Takes a CGWindowID, desired width/height, and a C function pointer callback
- Uses `SCShareableContent.excludingDesktopWindows(false, onScreenWindowsOnly: true)` to find the window
- Creates `SCContentFilter(desktopIndependentWindow: window)` for window-specific capture
- Configures `SCStreamConfiguration` with:
  - `pixelFormat = kCVPixelFormatType_32BGRA` (required -- SkiaSharp expects BGRA, NOT default YUV)
  - `captureResolution = .best` (Retina quality)
  - `showsCursor = false` (clean game capture)
  - `width` and `height` from parameters
- Calls `SCScreenshotManager.captureImage(contentFilter:configuration:)` (macOS 14+ API, single-frame, no stream lifecycle)
- **CGImage to PNG conversion (CRITICAL -- Mac Catalyst compatible):**
  Do NOT use `NSBitmapImageRep` -- it requires AppKit which is unavailable on Mac Catalyst.
  Instead, use ImageIO's `CGImageDestination` to convert CGImage to PNG data:
  ```swift
  import ImageIO
  import UniformTypeIdentifiers

  func cgImageToPNGData(_ cgImage: CGImage) -> Data? {
      let data = NSMutableData()
      guard let dest = CGImageDestinationCreateWithData(data as CFMutableData, UTType.png.identifier as CFString, 1, nil) else {
          return nil
      }
      CGImageDestinationAddImage(dest, cgImage, nil)
      guard CGImageDestinationFinalize(dest) else {
          return nil
      }
      return data as Data
  }
  ```
  This uses CoreGraphics + ImageIO which are available on ALL Apple platforms including Mac Catalyst.
- Calls the callback with PNG bytes pointer and length, OR (nil, 0) on failure
- Wrap all SCK calls in `Task { @MainActor in ... }` to ensure the NSRunLoop is active (pitfall #2 from research: SCK hangs without run loop)
- Wrap SCK code in `if #available(macCatalyst 18.2, macOS 14.0, *)` guard (pitfall #1: crash on older Mac Catalyst)

**Important implementation notes:**
- Import: Foundation, ScreenCaptureKit, CoreGraphics, ImageIO, UniformTypeIdentifiers
- Do NOT import AppKit -- it is unavailable on Mac Catalyst and will cause compilation failure
- The `@_cdecl` attribute makes functions callable via `DllImport` from C# without any ObjC messaging
- The callback pattern lets C# use `TaskCompletionSource` to bridge to async/await
- Do NOT use SCStream (overkill for 30-second interval captures -- research anti-pattern)
- Do NOT use `@objc` or ObjC class patterns -- @_cdecl is simpler for this use case
- Keep the file focused: two exported functions + one helper, no unnecessary abstractions

Create `src/GaimerDesktop/NativeHelpers/GaimerScreenCapture/build-xcframework.sh` (make executable with chmod +x).

This script uses **xcodebuild** as the primary build tool (NOT swiftc directly). The research doc recommends starting with an Xcode project / Package.swift, and xcodebuild handles Mac Catalyst framework resolution, linking, and code signing correctly.

The script must:

1. Set `-e` for fail-fast, define SCRIPT_DIR (dirname of script), BUILD_DIR, FRAMEWORK_NAME=GaimerScreenCapture
2. Clean previous build artifacts (`rm -rf $BUILD_DIR`)
3. **Build the Swift Package for Mac Catalyst using xcodebuild:**
   ```bash
   xcodebuild build \
     -scheme GaimerScreenCapture \
     -destination "generic/platform=macOS,variant=Mac Catalyst" \
     -derivedDataPath "$BUILD_DIR/derived" \
     -configuration Release \
     BUILD_LIBRARY_FOR_DISTRIBUTION=YES \
     SUPPORTS_MACCATALYST=YES
   ```
   This builds the dynamic library from Package.swift for the Mac Catalyst destination.

4. **Locate the built framework or dylib** in the derived data:
   ```bash
   BUILT_FRAMEWORK=$(find "$BUILD_DIR/derived" -name "GaimerScreenCapture.framework" -type d | head -1)
   # If xcodebuild produces a .framework, use it directly
   # If it produces a bare dylib, create the framework structure manually:
   #   mkdir -p $BUILD_DIR/GaimerScreenCapture.framework/
   #   cp <dylib> $BUILD_DIR/GaimerScreenCapture.framework/GaimerScreenCapture
   #   Create Info.plist with CFBundleIdentifier=com.5dof.gaimer.screencapture
   ```

5. **Create the xcframework:**
   ```bash
   xcodebuild -create-xcframework \
     -framework "$BUILT_FRAMEWORK" \
     -output "$BUILD_DIR/GaimerScreenCapture.xcframework"
   ```

6. **Copy the xcframework to the MAUI project:**
   ```bash
   DEST="$SCRIPT_DIR/../../GaimerDesktop/Platforms/MacCatalyst/"
   mkdir -p "$DEST"
   rm -rf "$DEST/GaimerScreenCapture.xcframework"
   cp -R "$BUILD_DIR/GaimerScreenCapture.xcframework" "$DEST"
   ```

7. Print success message with output path

**IMPORTANT NOTES:**
- xcodebuild can build Swift Packages directly (no .xcodeproj needed) when given a scheme name matching the package target
- If xcodebuild cannot resolve the scheme from Package.swift alone, generate an Xcode project first: `swift package generate-xcodeproj --package-path "$SCRIPT_DIR"` and then use `-project "$SCRIPT_DIR/GaimerScreenCapture.xcodeproj"`
- BUILD_LIBRARY_FOR_DISTRIBUTION=YES is required for xcframework compatibility (generates module stability headers)
- The dev machine has Xcode 26.2 and Swift 6.2.3 -- both support Mac Catalyst
