  <action>
Run the full build-and-launch sequence from CLAUDE.md:

```bash
dotnet build src/GaimerDesktop/GaimerDesktop/GaimerDesktop.csproj -f net8.0-maccatalyst -p:EnableCodeSigning=false && \
rm -rf /tmp/GaimerDesktop.app && \
ditto --norsrc src/GaimerDesktop/GaimerDesktop/bin/Debug/net8.0-maccatalyst/maccatalyst-x64/GaimerDesktop.app /tmp/GaimerDesktop.app && \
codesign --force --deep --sign - /tmp/GaimerDesktop.app && \
open /tmp/GaimerDesktop.app
```

Before launching, also open Apple Chess.app so it is available as a capture target:
```bash
open -a Chess
```

Watch the debug output for:
- `[Capture] Capture method: ScreenCaptureKit (GPU-capable)` -- confirms SCK is active
- `[Capture] Screen Recording preflight: granted` -- confirms permission
- Any `[Capture] SCK error:` or `[Capture] SCK capture timed out` messages indicating problems

If the build fails, check the build output for:
- NativeReference linking errors (xcframework not found or incompatible)
- P/Invoke signature mismatches
- Missing framework dependencies

If the app crashes on launch, check:
- Whether SCK availability check itself crashes (the try/catch should prevent this, but check)
- Whether the xcframework binary is for the correct architecture

- No crashes or freezes during capture
