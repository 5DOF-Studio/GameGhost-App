## Shell.Current Strategy
Methods that hit `Shell.Current` (navigation) will encounter null in tests. Strategy:
- Test business logic UP TO the Shell call
- Verify all mock interactions before Shell code
- Shell.Current null → method returns early or throws NRE → wrap navigation calls in null-check or test with try/catch
- Navigation correctness is deferred to manual/integration testing
