# Plan 09-02: Dual Chess Tools + ToolExecutor Integration

## Goal
Replace the Lichess cloud eval `get_best_move` tool with a Stockfish-powered `analyze_position_engine` tool. Add a new `analyze_position_strategic` tool for LLM-based strategic analysis. Update ToolDefinitions, ToolExecutor, and SessionManager to register both tools. Wire IStockfishService into DI.

## Dependencies
- 09-01 (IStockfishService must exist)

## Design Decision: Two Tools, Not One Dispatcher

The brain LLM chooses which tool to call based on system prompt guidance:
- **analyze_position_engine**: "What's the best move?" — Stockfish, authoritative, mathematical
- **analyze_position_strategic**: "What's the plan?" — LLM self-analysis, strategic themes, explanations

The brain can call both in sequence for critical positions (engine truth + strategic explanation).

### `Services/Brain/ToolExecutor.cs` — Add new tool handlers

**Constructor changes:**
- Add `IStockfishService _stockfishService` parameter
- Keep `_lichessClient` as fallback (if Stockfish not installed, fall back to Lichess)

**New tool dispatch entries:**
```csharp
"analyze_position_engine" => await ExecuteAnalyzePositionEngineAsync(argumentsJson, ct),
"analyze_position_strategic" => await ExecuteAnalyzePositionStrategicAsync(argumentsJson, ct),
"get_best_move" => await ExecuteGetBestMoveAsync(argumentsJson, ct),  // keep legacy fallback
```

**`ExecuteAnalyzePositionEngineAsync` implementation:**
1. Parse `fen`, `depth` (default 25), `num_lines` (default 3) from arguments
2. Validate FEN via `FenValidator.IsValid()` — if invalid, return `{ status: "fen_invalid", errors: [...], hint: "Re-examine the board" }`
3. Call `_stockfishService.AnalyzePositionAsync(fen, new AnalysisOptions(MoveTimeMs: 1000, MaxDepth: depth, MultiPv: numLines))`
4. If Stockfish not ready: fall back to `ExecuteGetBestMoveAsync` (Lichess)
5. Return JSON with: status, bestMove, evaluation (cp or mate), assessment, depth, candidates (each with move, eval, continuation line)
6. Include human-readable assessment text from ClassifyEvaluation

**Return schema:**
```json
{
  "status": "success",
  "bestMove": "e7e5",
  "evaluation": { "type": "cp", "value": 22 },
  "assessment": "Equal",
  "signal": "neutral",
  "depth": 20,
  "candidates": [
    { "move": "e7e5", "eval": 22, "line": "e7e5 g1f3 b8c6 f1b5 g8f6", "assessment": "Equal" },
    { "move": "c7c5", "eval": 32, "line": "c7c5 g1f3 b8c6 f1b5 e7e6", "assessment": "Equal" },
    { "move": "c7c6", "eval": 27, "line": "c7c6 d2d4 d7d5 e4e5 c6c5", "assessment": "Equal" }
  ]
}
```

**`ExecuteAnalyzePositionStrategicAsync` implementation:**
1. Parse optional `focus` and `player_color`
2. Build a focused strategic analysis prompt
3. Call `_openRouterClient.ChatCompletionAsync()` with the worker model + strategic prompt
4. Return structured JSON with themes, plans, assessment

**Strategic prompt template:**
```
You are a chess analysis engine. Analyze this position strategically.
Focus: {focus}
Player color: {player_color}

Provide:
1. Key strategic themes (2-3 bullet points)
2. Piece activity assessment (which pieces are active/passive)
3. Pawn structure evaluation
4. Recommended plan (1-2 sentences)
5. King safety assessment

Keep analysis concise — this feeds a voice coaching agent.
```
