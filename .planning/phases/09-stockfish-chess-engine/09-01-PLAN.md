# Plan 09-01: Stockfish Engine Service + UCI Wrapper

## Goal
Build `IStockfishService` — a managed wrapper around the Stockfish binary that spawns it as a child process, communicates via UCI protocol over stdin/stdout, and provides async analysis results. Includes on-demand download with progress reporting.

## Dependencies
- None (foundation layer)

## Files to Create

### `Services/Chess/IStockfishService.cs`
```csharp
public interface IStockfishService : IDisposable
{
    bool IsReady { get; }
    bool IsInstalled { get; }
    Task<bool> EnsureInstalledAsync(IProgress<double>? progress = null, CancellationToken ct = default);
    Task StartAsync(CancellationToken ct = default);
    Task StopAsync();
    Task<EngineAnalysis> AnalyzePositionAsync(string fen, AnalysisOptions? options = null, CancellationToken ct = default);
}

public record AnalysisOptions(int MoveTimeMs = 1000, int MaxDepth = 25, int MultiPv = 1);

public record EngineAnalysis(
    string BestMove,
    string? PonderMove,
    int Depth,
    IReadOnlyList<EngineVariation> Variations,
    long Nodes,
    int TimeMs);

public record EngineVariation(
    int MultiPvIndex,
    string Move,           // UCI format: e2e4
    int? CentipawnEval,    // null if mate
    int? MateIn,           // null if centipawn
    string[] PrincipalVariation,  // full line in UCI
    int Depth);
```

### `Services/Chess/StockfishService.cs`
Core implementation:
- **Process management:** Spawn Stockfish via `System.Diagnostics.Process` with redirected stdin/stdout
- **UCI handshake:** Send `uci`, wait for `uciok`. Send `setoption name Threads value 2`, `setoption name Hash value 128`. Send `isready`, wait for `readyok`.
- **Analysis:** Send `position fen {fen}`, then `go movetime {ms} depth {max}`. Parse `info depth ... multipv ... score cp/mate ... pv ...` lines. Return when `bestmove` line received.
- **Cancellation:** Send `stop` command to abort analysis mid-search
- **Lifecycle:** `StartAsync` spawns process, `StopAsync` sends `quit` + Kill, `Dispose` cleanup
- Async stdout reading via dedicated `Task.Run` loop with `ReadLineAsync`
- **CRITICAL:** Never synchronously read both stdout and stderr (deadlock). Read stderr via `BeginErrorReadLine` for logging only.

### `Services/Chess/StockfishDownloader.cs`
- Platform detection: `RuntimeInformation.IsOSPlatform` + `OSArchitecture`
- Download URLs: configurable (default: own CDN/GitHub releases)
- Storage: `Path.Combine(Environment.GetFolderPath(SpecialFolder.ApplicationData), "com.gaimer.witnessdesktop", "engines", binaryName)`
- SHA256 verification after download
- `chmod +x` on macOS via `Process.Start("chmod", "+x path")`
- `IProgress<double>` reporting during streaming download
- Temp file + atomic rename pattern (download to `.tmp`, verify, move)

### `Services/Chess/MockStockfishService.cs`
- Always `IsReady = true`, `IsInstalled = true`
- Returns canned analysis results for known FEN positions
- Falls back to reasonable defaults (+30cp, bestMove="e2e4") for unknown positions
- For TDD: deterministic, no process spawning, no network

### `Services/Chess/FenValidator.cs`
- Static `bool IsValid(string fen, out List<string> errors)`
- Validates: 8 ranks separated by `/`, valid piece chars (`KQRBNPkqrbnp`), digit sum per rank = 8
- Validates: exactly 1 white king, exactly 1 black king
- Validates: max 8 pawns per side, no pawns on rank 1 or 8
- Validates: side to move is `w` or `b`
- Validates: castling rights from `KQkq-` chars
- Does NOT validate legality (check detection) — that requires move generation

## Notes
- Stockfish binary is ~108MB uncompressed, downloaded on-demand (not bundled)
- GPL3 license: separate process = aggregation, not derivative work
- Include `STOCKFISH_LICENSE.txt` with GPLv3 text alongside binary
- Recommended config: Threads=2, Hash=128MB, MultiPV=1 default
- `go movetime 1000` reaches depth 17 with MultiPV=3 on i7-9750H, depth 19-23 with MultiPV=1 on M1/M2

## Download Process (Verified on Dev Machine)
- GitHub API asset names (exact): `stockfish-macos-m1-apple-silicon.tar`, `stockfish-macos-x86-64-bmi2.tar`, `stockfish-windows-x86-64-bmi2.zip`
- Archive structure: `stockfish/stockfish-{variant}` — binary is nested, NOT at root
- CPU feature detection: `sysctl -n hw.optional.bmi2` on macOS, fallback chain: bmi2 > avx2 > sse41-popcnt > generic
- CRITICAL: pipe/heredoc input causes premature exit (stdin EOF). MUST use Process.RedirectStandardInput (stdin stays open).
- Dev binary location: `~/Library/Application Support/com.gaimer.witnessdesktop/engines/stockfish`
