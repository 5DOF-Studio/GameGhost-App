    **MainViewModel.cs:**

    This is the most complex modification. The ViewModel already has FAB state (IsFabActive, FabCardVariant, etc.). Ghost mode integration means:

    1. **Inject IGhostModeService:** Add to constructor parameters alongside existing services. Store as `private readonly IGhostModeService _ghostModeService;`

    2. **Modify ToggleFab():** Currently just toggles `IsFabActive = !IsFabActive`. Change to:
       ```csharp
       private async void ToggleFab()
       {
           if (!_ghostModeService.IsSupported)
           {
               // Fallback: just toggle FAB overlay in MAUI window (existing behavior)
               IsFabActive = !IsFabActive;
               return;
           }

           if (_ghostModeService.IsGhostModeActive)
           {
               // Exit ghost mode
               await _ghostModeService.ExitGhostModeAsync();
               IsFabActive = false;
           }
           else
           {
               // Enter ghost mode
               IsFabActive = true;
               _ghostModeService.SetFabState(active: true, connected: IsConnected);

               // Resolve and set agent image path
               if (SelectedAgent?.PortraitImage != null)
               {
                   var imageName = SelectedAgent.PortraitImage
                       .Replace(".png", "")
                       .Replace(".jpg", "")
                       .ToLowerInvariant();
                   var path = Foundation.NSBundle.MainBundle.PathForResource(imageName, "png");
                   if (!string.IsNullOrEmpty(path))
                       _ghostModeService.SetAgentImage(path);
               }

               await _ghostModeService.EnterGhostModeAsync();
           }
       }
       ```
       Wrap the Foundation.NSBundle call in `#if MACCATALYST` or use a conditional approach.

    3. **Subscribe to ghost mode events:** In the constructor or initialization:
       ```csharp
       _ghostModeService.FabTapped += (s, e) =>
       {
           MainThread.BeginInvokeOnMainThread(() => ToggleFab());
       };
       _ghostModeService.CardDismissed += (s, e) =>
       {
           MainThread.BeginInvokeOnMainThread(() =>
           {
               FabCardVariant = FabCardVariant.None;
           });
       };
       ```
       The FabTapped callback fires from native code (main thread) but ViewModel property changes need to be on the MAUI main thread. Use MainThread.BeginInvokeOnMainThread for safety.

    4. **Forward AI events to ghost mode:** In the existing TextReceived handler (where `FabCardVariant = FabCardVariant.Text` is set), add:
       ```csharp
       if (_ghostModeService.IsGhostModeActive)
       {
           _ghostModeService.ShowCard(FabCardVariant.Text, "Dross", messageText, null);
       }
       ```

    5. **Forward voice state to ghost mode:** In the existing UpdateFabVoiceState method (where FabCardVariant.Voice is set), add:
       ```csharp
       if (_ghostModeService.IsGhostModeActive && FabCardVariant == FabCardVariant.Voice)
       {
           _ghostModeService.ShowCard(FabCardVariant.Voice, null, "is talking...", null);
       }
       else if (_ghostModeService.IsGhostModeActive && FabCardVariant == FabCardVariant.None)
       {
           _ghostModeService.DismissCard();
       }
       ```

    6. **Sync FAB connected state:** In the existing OnPropertyChanged for IsConnected (where IsFabEnabled is raised), add:
       ```csharp
       if (_ghostModeService.IsGhostModeActive)
       {
           _ghostModeService.SetFabState(IsFabActive, IsConnected);
       }
       ```

    7. **Clean exit on disconnect:** In the existing disconnect handler that resets IsFabActive, add:
       ```csharp
       if (_ghostModeService.IsGhostModeActive)
       {
           await _ghostModeService.ExitGhostModeAsync();
       }
       ```

    IMPORTANT: Read MainViewModel.cs fully before making changes. The ToggleFab method, TextReceived handler, UpdateFabVoiceState, and disconnect handler are already partially stubbed. Integrate ghost mode calls into the existing flow without breaking current dashboard-mode FAB behavior.

    IMPORTANT: The ghost mode service calls should be guarded with `_ghostModeService.IsGhostModeActive` so they only fire when actually in ghost mode. Dashboard mode should continue to work exactly as before.
