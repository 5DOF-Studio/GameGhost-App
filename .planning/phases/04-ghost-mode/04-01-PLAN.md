  <action>
    Create the complete GaimerGhostMode Swift package. This builds for **macOS** (NOT Mac Catalyst) because it needs `import AppKit` for NSPanel.

    **Package.swift:** Target `.macOS(.v13)`. Dynamic library. Link AppKit, SwiftUI, CoreGraphics frameworks. Follow the exact structure of the existing GaimerScreenCapture/Package.swift but change platform from macCatalyst to macOS.

    **GhostPanel.swift:** NSPanel subclass with:
    - `styleMask: [.borderless, .nonactivatingPanel]`
    - `isOpaque = false`, `backgroundColor = .clear`, `hasShadow = false`
    - `level = .floating`, `isFloatingPanel = true`
    - `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`
    - `isExcludedFromWindowsMenu = true`
    - `isReleasedWhenClosed = false`
    - `hidesOnDeactivate = false`
    - Override `canBecomeKey` to return `true`, `canBecomeMain` to return `false`
    - A method `setupContent(_ rootView: some View)` that wraps the SwiftUI root view in a ClickThroughHostingView and sets it as contentView

    **ClickThroughHostingView** (inside GhostPanel.swift or its own section): NSHostingView subclass that overrides `hitTest(_:)`:
    - If `super.hitTest(point)` returns `nil` -> return `nil` (click falls through)
    - If `super.hitTest(point)` returns `self` -> return `nil` (transparent area, click falls through)
    - Otherwise return the hit view (interactive element like FAB or card)

    CRITICAL: Do NOT set `ignoresMouseEvents = true` on the panel. That would make everything click-through including interactive elements.

    **GhostPanelContentView.swift:** SwiftUI root view wrapping FabButton + EventCardView in a VStack/ZStack layout. Uses @ObservedObject or @StateObject with a shared GhostPanelState class (ObservableObject) that holds:
    - `agentImagePath: String?`
    - `isFabActive: Bool`
    - `isFabConnected: Bool`
    - `cardVariant: Int32` (0=None, 1=Voice, 2=Text, 3=TextWithImage)
    - `cardTitle: String?`
    - `cardText: String?`
    - `cardImagePath: String?`
    - `onFabTap: (() -> Void)?`
    - `onCardDismiss: (() -> Void)?`

    The view should position the FAB at bottom-right of the panel, and event cards above the FAB. The panel background must be `.clear` (fully transparent). Only the FAB circle and card rectangles should have visible backgrounds.

    **FabView.swift:** SwiftUI view matching the existing XAML FabOverlayView design:
    - Circular button (56pt diameter)
    - Shows agent portrait image (loaded from file path via NSImage(contentsOfFile:))
    - Yellow glow ring (2pt stroke + 12pt shadow) when connected
    - Dims to 0.3 opacity when NOT connected
    - Shows X icon when active (in ghost mode)
    - Dark background circle (#1E1E2E)
    - Calls onTap closure when tapped

    **EventCardView.swift:** SwiftUI view with three variants:
    - **Voice (1):** Agent avatar circle + "is talking..." label + simple animated dots
    - **Text (2):** Rounded rect card with title pill (cyan background) + text body. Dark semi-transparent background (#1E1E2E at 0.95 opacity). Max width 280pt.
    - **TextWithImage (3):** Same as Text but with an image below the text (loaded from file path)
    - All cards: rounded corners (12pt), subtle shadow, tap-to-dismiss gesture
    - Auto-dismiss: Use SwiftUI `.task` with `Task.sleep(nanoseconds:)` for 8-second auto-dismiss. Call onDismiss when timer fires or user taps.

    **GaimerGhostMode.swift:** The @_cdecl exports file. This is the C API surface consumed by C# DllImport. Create a file-level `var sharedPanel: GhostPanel?` and `var sharedState: GhostPanelState?` to hold the singleton panel instance.

    All 14 @_cdecl functions as specified in the research doc (Pattern 5):
    - `ghost_panel_create() -> Bool` — Creates GhostPanel + GhostPanelState + sets up content view. Uses `DispatchQueue.main.async` for all AppKit work (do NOT use @MainActor to avoid deadlock). Returns true on success.
    - `ghost_panel_destroy()` — Closes panel, nils references
    - `ghost_panel_show()` — `DispatchQueue.main.async { panel.orderFront(nil) }`
    - `ghost_panel_hide()` — `DispatchQueue.main.async { panel.orderOut(nil) }`
    - `ghost_panel_set_agent_image(pathPtr)` — Updates sharedState.agentImagePath
    - `ghost_panel_set_fab_active(active)` — Updates sharedState.isFabActive
    - `ghost_panel_set_fab_connected(connected)` — Updates sharedState.isFabConnected
    - `ghost_panel_show_card(variant, titlePtr, textPtr, imagePathPtr)` — Updates card state fields
    - `ghost_panel_dismiss_card()` — Sets cardVariant to 0
    - `ghost_panel_set_fab_tap_callback(callback)` — Stores C function pointer, wires to sharedState.onFabTap
    - `ghost_panel_set_card_dismiss_callback(callback)` — Stores C function pointer, wires to sharedState.onCardDismiss
    - `ghost_panel_set_position(x, y)` — `DispatchQueue.main.async { panel.setFrameOrigin(...) }`
    - `ghost_panel_set_size(width, height)` — `DispatchQueue.main.async { panel.setContentSize(...) }`

    CRITICAL: Use `DispatchQueue.main.async` (NOT `@MainActor`) for all UI operations to avoid deadlock when C# side blocks with Task.Wait(). This was a lesson from Phase 03.

    CRITICAL: For `ghost_panel_create`, the function must dispatch to main queue and return immediately. The creation is async fire-and-forget. The C# side should call `ghost_panel_show` after a brief delay or use a creation callback if synchronous creation confirmation is needed. Alternatively, if creation is fast enough, use `DispatchQueue.main.sync` ONLY for create (since it's called once at init, not from the main thread).

    CRITICAL: String parameters (UnsafePointer<CChar>) must be converted to Swift String immediately (inside the @_cdecl function) before dispatching to main queue, because the pointer may be freed by .NET after the @_cdecl returns.

    CRITICAL: Callback function pointers (`@convention(c) () -> Void`) stored from set_fab_tap_callback and set_card_dismiss_callback must be invoked from ANY thread (they will be invoked from the main thread when user taps). The C# side pins these with GCHandle.
