# Plan 11-01: Agent Personality Architecture — Structured Sections + Design File Scaffold

## Objective
Restructure Agent.cs personality system from monolithic SystemInstruction strings into composed sections (SOUL, STYLE, BEHAVIOR, SITUATIONS, ANTI-PATTERNS). Create the design file directory structure for human-authored personality content. Establish the compile-time composition pattern.

## Rationale
Currently, Agent.SystemInstruction is a single hand-written string per agent. The ChessToolGuidance pattern (const string interpolated into SystemInstruction) already proves the composition model works. This plan extends it to all personality dimensions so that:
1. Personality content is authored in separate, human-readable design files
2. Const strings in Agent.cs compose the final SystemInstruction at compile time
3. The same personality source feeds voice, brain, and chat injection points (plans 02-03)

## Tasks

### Task 1: Create design file scaffold
Create directory structure for per-agent personality design files:
```
GAIMER/GameGhost/gaimer_spec_docs/agents/
  leroy/
    SOUL.md           — placeholder with section headings from BUILD template
    STYLE.md          — placeholder with section headings
    BEHAVIOR.md       — placeholder with section headings
    SITUATIONS.md     — placeholder with section headings
    ANTI-PATTERNS.md  — placeholder with section headings
    EXAMPLES.md       — good-outputs + bad-outputs sections
  wasp/
    (same structure)
```
These are human-editable design docs. Code reads from const strings, not from these files at runtime.

**Files:** 12 new markdown files (6 per agent)

### Task 2: Add personality section properties to Agent.cs
Add optional personality section strings to Agent class:
```csharp
// Personality composition sections (compiled from design files)
public string? SoulBlock { get; init; }
public string? StyleBlock { get; init; }
public string? BehaviorBlock { get; init; }
public string? SituationsBlock { get; init; }
public string? AntiPatternsBlock { get; init; }
```

Add a computed property that composes them:
```csharp
/// <summary>
/// Full personality prompt composed from structured sections.
/// Falls back to legacy SystemInstruction if sections aren't populated.
/// </summary>
public string ComposedPersonality => BuildComposedPersonality();
```

The BuildComposedPersonality method concatenates non-null sections with clear headers:
```
[WHO YOU ARE]
{SoulBlock}

[HOW YOU TALK]
{StyleBlock}

[HOW YOU BEHAVE]
{BehaviorBlock}

[SITUATIONAL MODES]
{SituationsBlock}

[NEVER DO]
{AntiPatternsBlock}
```

If no sections are populated, falls back to SystemInstruction (backward compatible).

**Files:** `Models/Agent.cs`

### Task 3: Create placeholder const strings for Leroy and Wasp
Add const string blocks in Agent.cs for each section per agent. Initially these are minimal placeholders (copied from current SystemInstruction content, split into sections). They will be replaced with rich content from the builder questionnaire in Plan 11-04/11-05.

For Leroy:
- `LeroySoul` — extract identity paragraph from current SystemInstruction
- `LeroyStyle` — extract voice output rules from ChessToolGuidance
- `LeroyBehavior` — extract behavior rules from current SystemInstruction
- `LeroySituations` — currently empty placeholder
- `LeroyAntiPatterns` — currently empty placeholder

For Wasp:
- Same pattern, distinct personality extract from current Wasp SystemInstruction

Wire these into the new section properties on each Agent instance. Keep SystemInstruction as-is for backward compatibility until plan 02-03 switch consumers to ComposedPersonality.

**Files:** `Models/Agent.cs`

### Task 4: Add brain personality prefix property
Add a compact personality prefix for brain/analytical prompts:
```csharp
/// <summary>
/// Compact personality prefix for brain prompts (~200 tokens).
/// Brain gets identity + analytical style, NOT full voice STYLE.
/// </summary>
public string? BrainPersonalityPrefix { get; init; }
```

This is a shorter version of SOUL + analytical style, designed for the OpenRouter brain context window. Voice gets the full ComposedPersonality; brain gets BrainPersonalityPrefix.

**Files:** `Models/Agent.cs`
