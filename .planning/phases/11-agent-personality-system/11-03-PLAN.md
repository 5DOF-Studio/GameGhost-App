# Plan 11-03: OpenRouterBrainService Agent-Awareness — Personality in Brain Prompts

## Objective
Inject the selected agent's BrainPersonalityPrefix into OpenRouterBrainService prompts so brain analysis output carries agent personality, not generic "gaming AI analyst" text. This closes the biggest personality gap: brain output that flows through Channel<BrainResult> to voice currently has zero personality.

## Rationale
Currently in OpenRouterBrainService:
- `SubmitImageAsync` line 78: `"You are a gaming AI analyst. Analyze this game screenshot..."`
- `SubmitQueryAsync` line 180-185: generic system context with no personality

Brain output flows through Channel → BrainEventRouter → Voice (via SendContextualUpdateAsync). If the brain writes generic text, the voice reads generic text. Injecting personality at the brain level means the analysis *itself* is in-character.

## Current Flow
```
Capture → Brain ("You are a gaming AI analyst") → Channel → Router → Voice reads generic text
```

## Target Flow
```
Capture → Brain ("You are Leroy. You're sharp, confident...Analyze this...") → Channel → Router → Voice reads in-character text
```

## Tasks

### Task 1: Pass Agent reference to OpenRouterBrainService
Add agent context to the brain service. Options:
- **Option A:** Pass Agent via ISessionManager (SessionContext already has GameType, can add AgentKey)
- **Option B:** Add Agent parameter to SubmitImageAsync/SubmitQueryAsync
- **Option C:** Set agent on the service instance when agent is selected

Option A is cleanest — SessionContext gains an `AgentKey` string, and the brain reads `Agents.GetByKey(session.AgentKey)?.BrainPersonalityPrefix`. This requires no interface changes.

**Files:** `Models/SessionContext.cs` (add AgentKey), `Services/Brain/OpenRouterBrainService.cs`

### Task 2: Inject BrainPersonalityPrefix into SubmitImageAsync
Replace the hardcoded prompt:
```csharp
// Before
var contextPrompt = $"You are a gaming AI analyst. Analyze this game screenshot...";

// After
var personalityPrefix = GetBrainPersonality();
var contextPrompt = $"{personalityPrefix} Analyze this game screenshot and provide tactical insights. Current game: {gameType}. {context}";
```

Where `GetBrainPersonality()` reads the current agent's BrainPersonalityPrefix, falling back to the generic prompt if not available.

**Files:** `Services/Brain/OpenRouterBrainService.cs`

### Task 3: Inject personality into SubmitQueryAsync
Same pattern for the query path — inject personality prefix into the system message:
```csharp
var personalityPrefix = GetBrainPersonality();
var systemContext = $"{personalityPrefix} Game state: {state}. Game: {game}. Intent: {intent}. {summary}";
```

**Files:** `Services/Brain/OpenRouterBrainService.cs`

### Task 4: Set AgentKey on session when agent is selected
In MainViewModel (or AgentSelectionViewModel), when the user selects an agent:
```csharp
_sessionManager.Context.AgentKey = agent.Key;
```

This ensures the brain service always knows which agent is active.

**Files:** `ViewModels/MainViewModel.cs` or `ViewModels/AgentSelectionViewModel.cs`

### Task 5: Update tests
- Test SubmitImageAsync uses agent personality when AgentKey is set
- Test SubmitImageAsync uses generic prompt when AgentKey is null/empty
- Test SubmitQueryAsync injects personality prefix
- Test AgentKey is set on session context during agent selection

**Files:** `GaimerDesktop.Tests/` (existing OpenRouterBrainService tests + new)

## Design Decision: ToolExecutor stays personality-free
ToolExecutor prompts (`"You are a chess analysis engine"`, `"You are a gaming knowledge assistant"`) remain generic. Tools produce objective data — personality is applied by the brain when it synthesizes tool results into its final response. This is correct and intentional.
