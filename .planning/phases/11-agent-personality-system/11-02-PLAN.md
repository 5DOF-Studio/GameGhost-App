# Plan 11-02: ChatPromptBuilder Agent-Awareness — Replace Hardcoded Dross

## Objective
Make ChatPromptBuilder agent-aware by injecting the selected agent's personality instead of the hardcoded "Dross" identity. The text chat prompt should reflect who the user is talking to (Leroy, Wasp, or Derek), not a generic Dross identity.

## Rationale
Currently, ChatPromptBuilder.CoreIdentity is hardcoded as:
> "You are Dross, the AI copilot built into Gaimer..."

This string is used for ALL agents regardless of which one is selected. The agent's SystemInstruction only reaches the voice provider — the text chat brain has no awareness of agent identity.

## Current Flow
```
Agent Selection → MainViewModel stores _selectedAgent
ChatPromptBuilder.BuildSystemPrompt(session, tools) → hardcoded Dross
```

## Target Flow
```
Agent Selection → MainViewModel stores _selectedAgent
ChatPromptBuilder.BuildSystemPrompt(session, tools, agent) → agent.ComposedPersonality (text variant)
```

## Tasks

### Task 1: Update IChatPromptBuilder interface
Add agent parameter to BuildSystemPrompt:
```csharp
string BuildSystemPrompt(SessionContext session, IReadOnlyList<ToolDefinition> tools, Agent? agent = null);
```

The null default maintains backward compatibility for any callers not yet updated.

**Files:** `Services/IChatPromptBuilder.cs` (if interface exists separately), `Services/ChatPromptBuilder.cs`

### Task 2: Replace CoreIdentity with agent personality
In ChatPromptBuilder.BuildSystemPrompt:
- If agent is provided and has ComposedPersonality, use the text-adapted version
- If agent is null, fall back to current CoreIdentity (Dross)

Create a private method `BuildAgentIdentity(Agent? agent)` that:
1. Uses agent.SoulBlock if available (identity section)
2. Adapts STYLE for text medium (not voice) — replaces voice-specific rules with TextMediumRules
3. Falls back to CoreIdentity const if no agent provided

The key insight: voice gets full ComposedPersonality (voice-optimized STYLE), chat gets SOUL + text-adapted STYLE + BEHAVIOR. The SITUATIONS and ChessToolGuidance are voice-only.

**Files:** `Services/ChatPromptBuilder.cs`

### Task 3: Wire agent through to ChatPromptBuilder callers
Find all callers of BuildSystemPrompt and pass the current agent:
- MainViewModel or wherever ChatPromptBuilder is used
- Pass `_selectedAgent` or equivalent

**Files:** Dependent on caller locations (likely `ViewModels/MainViewModel.cs` or `Services/BrainEventRouter.cs`)

### Task 4: Update ChatPromptBuilder tests
- Test that BuildSystemPrompt with Leroy agent produces Leroy identity (not Dross)
- Test that BuildSystemPrompt with null agent falls back to Dross
- Test that text-adapted style uses TextMediumRules (not voice rules)
- Verify session context and tool sections are unchanged

**Files:** `GaimerDesktop.Tests/ChatPromptBuilderTests.cs`
