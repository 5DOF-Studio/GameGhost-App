# Plan 11-06: Integration Verification — Personality Persistence Across All LLM Touchpoints

## Objective
Verify that agent personality persists across all three LLM touchpoints (voice provider, brain service, chat prompt builder) and that the end-to-end pipeline produces in-character output from capture through to voice delivery.

## Prerequisites
- Plans 11-01 through 11-05 complete (architecture + both agent personalities composed)

## Tasks

### Task 1: Voice personality verification
Test that ComposedPersonality is correctly sent to voice providers:
- Gemini: verify system_instruction contains SOUL + STYLE + BEHAVIOR + SITUATIONS + ANTI-PATTERNS + ChessToolGuidance
- OpenAI: verify instructions field contains same composed content
- Verify VoiceGender is correctly applied (male for Leroy, female for Wasp)

### Task 2: Brain personality verification
Test that BrainPersonalityPrefix is injected into brain prompts:
- SubmitImageAsync: verify prompt starts with agent personality, not generic "gaming AI analyst"
- SubmitQueryAsync: verify system context includes personality prefix
- Verify ToolExecutor remains personality-free (objective analysis)

### Task 3: Chat personality verification
Test that ChatPromptBuilder produces agent-aware prompts:
- Leroy selected: identity section says Leroy, not Dross
- Wasp selected: identity section says Wasp
- No agent (edge case): falls back to Dross
- TextMediumRules applied (not voice STYLE)

### Task 4: Pipeline integration test
End-to-end verification:
- Select Leroy → capture frame → brain analyzes with Leroy personality → Channel → BrainEventRouter → voice receives in-character text
- Select Wasp → same pipeline → verify Wasp personality in brain output
- Switch agents → verify personality switches cleanly

### Task 5: Token budget verification
Count actual tokens in composed prompts:
- Voice SystemInstruction per agent (target: <1500 tokens)
- Brain personality prefix per agent (target: <250 tokens)
- Chat system prompt per agent (target: <800 tokens)

## Acceptance Criteria
- [ ] Voice providers receive full ComposedPersonality
- [ ] Brain service uses BrainPersonalityPrefix
- [ ] Chat prompt builder uses agent identity
- [ ] ToolExecutor remains personality-free
- [ ] Agent switching updates all three touchpoints
- [ ] Token budgets within limits
- [ ] All existing tests pass (zero regressions)
- [ ] Build succeeds on both net8.0 and maccatalyst
