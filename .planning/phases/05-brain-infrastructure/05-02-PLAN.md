Create `Services/Brain/OpenRouterClient.cs`. Namespace: `GaimerDesktop.Services.Brain`.

**Constructor:**
- Accept `HttpClient httpClient, string apiKey, string defaultModel`
- Set `httpClient.BaseAddress = new Uri("https://openrouter.ai/api/v1/")`
- Set `Authorization: Bearer {apiKey}` header
- Set `X-Title: Gaimer Desktop` header (OpenRouter recommends for dashboard tracking)
- Set `Content-Type: application/json` default header

**Public methods:**

1. `async Task<OpenRouterResponse> ChatCompletionAsync(OpenRouterRequest request, CancellationToken ct = default)`
   - POST to `chat/completions`
   - Serialize request using `JsonSerializer.Serialize(request, OpenRouterJsonContext.Default.OpenRouterRequest)`
   - Read response, deserialize to `OpenRouterResponse` using source-gen context
   - If HTTP status is not success: read body, throw `OpenRouterException` with status code + body
   - Return deserialized response

2. `async IAsyncEnumerable<OpenRouterStreamChunk> ChatCompletionStreamAsync(OpenRouterRequest request, [EnumeratorCancellation] CancellationToken ct = default)`
   - Force `request.Stream = true`
   - POST to `chat/completions` with `HttpCompletionOption.ResponseHeadersRead`
   - Read response stream line by line
   - Skip empty lines and `data: [DONE]` sentinel
   - Parse `data: {json}` lines into `OpenRouterStreamChunk` using source-gen context
   - Yield each chunk
   - Handle cancellation gracefully (dispose stream)

3. `OpenRouterRequest CreateImageAnalysisRequest(byte[] imageData, string contextPrompt, string? model = null)`
   - Helper to build a request with vision content
   - Create message with role "user" and content as list: [ContentPart text (contextPrompt), ContentPart image_url with base64 data URI]
   - Set model to `model ?? _defaultModel`
   - Set max_tokens to 1024
   - Return the built request

4. `OpenRouterRequest CreateToolCallRequest(List<OpenRouterMessage> messages, List<OpenRouterTool> tools, string? model = null)`
   - Helper to build a tool-calling request
   - Set model, messages, tools, tool_choice = "auto"
   - Return the built request

**OpenRouterException:**
- Custom exception class (inner class or separate) with `HttpStatusCode StatusCode` and `string ResponseBody` properties.

**Important implementation notes:**
- Do NOT use `response.EnsureSuccessStatusCode()` — read body first for error details
- Use `OpenRouterJsonContext.Default` for all serialization (no reflection)
- The client is stateless per-request; thread safety comes from HttpClient being thread-safe
- No retry logic in the client (that's the caller's responsibility)

Create `Services/Brain/ToolExecutor.cs`. Namespace: `GaimerDesktop.Services.Brain`.

**Constructor:**
- Accept `IWindowCaptureService captureService, ISessionManager sessionManager`
- Store as readonly fields

**Public method:**

`async Task<string> ExecuteToolAsync(string toolName, string argumentsJson, CancellationToken ct = default)`

Returns a JSON string result that gets sent back to the LLM as tool_call result.

**Tool implementations (switch on toolName):**

1. `"capture_screen"`:
   - Check `_captureService.IsCapturing`. If not, return `{"error": "Not currently capturing any window"}`
   - Trigger a single capture. Since IWindowCaptureService uses event-based capture (FrameCaptured event with timer), and we need a synchronous result, use a TaskCompletionSource:
     - Subscribe to FrameCaptured event
     - Wait with 5-second timeout
     - On frame received, convert to base64 string
     - Return `{"status": "captured", "image_base64": "...", "target": "{processName}"}`
   - If timeout, return `{"error": "Capture timed out"}`

2. `"get_game_state"`:
   - Read `_sessionManager.Context`
   - Return JSON with: `state` (InGame/OutGame), `gameId`, `gameType`, `connectorName`, `gameStartedAt`, `availableTools` (list of tool names from GetAvailableTools)

3. `"get_best_move"`:
   - For now, return `{"status": "not_available", "reason": "Chess engine integration pending"}` — this is a placeholder. The brain LLM can use its own analysis for move suggestions. Full engine integration is future work.

4. `"web_search"`:
   - Parse argumentsJson for `"query"` field
   - For now, return `{"status": "not_implemented", "reason": "Web search integration pending", "query": "{parsed_query}"}` — placeholder for future integration

5. Default (unknown tool):
   - Return `{"error": "Unknown tool", "tool_name": "{toolName}"}`

**Static helper:**

`static List<OpenRouterTool> GetAvailableToolDefinitions(ISessionManager sessionManager)`
- Maps existing `ToolDefinitions` (CaptureScreen, GetBestMove, GetGameState, WebSearch) to `OpenRouterTool` format
- Filter based on session state: if OutGame, exclude RequiresInGame tools
- For parameters, use JsonDocument.Parse to create the JSON schema. Keep schemas simple:
  - capture_screen: no parameters (empty object)
  - get_game_state: no parameters
  - get_best_move: no parameters
  - web_search: `{"type": "object", "properties": {"query": {"type": "string", "description": "Search query"}}, "required": ["query"]}`

**Important:**
- All tool results are JSON strings — the LLM will parse them
- Use System.Text.Json for all JSON construction (JsonSerializer or Utf8JsonWriter)
- Handle exceptions within each tool: catch and return `{"error": "{exception.Message}"}`
- Do NOT block the calling thread — all paths are async
