**IBrainService.Results:** Returns `_channel.Reader`

**IBrainService.SubmitImageAsync(byte[] imageData, string context, CancellationToken ct):**
1. Create a linked CTS from `_globalCts.Token` and `ct`
2. Increment `_activeTasks` via Interlocked
3. Fire-and-forget via `Task.Run`:
   ```
   try
   {
       var correlationId = Guid.NewGuid().ToString();

       // Build request with vision
       var request = _client.CreateImageAnalysisRequest(imageData, contextPrompt, _brainModel);

       // Add tool definitions if in-game
       var tools = ToolExecutor.GetAvailableToolDefinitions(_sessionManager);
       if (tools.Count > 0)
       {
           request.Tools = tools;
       }

       // Execute with multi-turn tool calling
       var messages = new List<OpenRouterMessage>(request.Messages);
       var response = await _client.ChatCompletionAsync(request, linkedCt);

       // Tool call loop (max MaxToolTurns iterations)
       int toolTurns = 0;
       while (response.Choices?.Count > 0
              && response.Choices[0].FinishReason == "tool_calls"
              && toolTurns < MaxToolTurns)
       {
           var assistantMsg = response.Choices[0].Message;
           messages.Add(assistantMsg);

           // Execute each tool call
           foreach (var toolCall in assistantMsg.ToolCalls ?? [])
           {
               var toolResult = await _toolExecutor.ExecuteToolAsync(
                   toolCall.Function.Name, toolCall.Function.Arguments, linkedCt);
               messages.Add(new OpenRouterMessage
               {
                   Role = "tool",
                   Content = toolResult,
                   ToolCallId = toolCall.Id
               });
           }

           // Send tool results back to LLM
           request.Messages = messages;
           request.Tools = tools; // Keep tools available
           response = await _client.ChatCompletionAsync(request, linkedCt);
           toolTurns++;
       }

       // Extract final response
       var finalContent = response.Choices?.FirstOrDefault()?.Message?.Content?.ToString();

       // Write result to channel
       await _channel.Writer.WriteAsync(new BrainResult
       {
           Type = BrainResultType.ImageAnalysis,
           AnalysisText = finalContent ?? "No analysis available",
           VoiceNarration = TruncateForVoice(finalContent, 200),
           Priority = BrainResultPriority.WhenIdle,
           CorrelationId = correlationId,
       }, linkedCt);
   }
   catch (OperationCanceledException) { /* Normal */ }
   catch (Exception ex)
   {
       Debug.WriteLine($"[Brain] Image analysis failed: {ex.Message}");
       await _channel.Writer.WriteAsync(new BrainResult
       {
           Type = BrainResultType.Error,
           AnalysisText = ex.Message,
           Priority = BrainResultPriority.Silent,
       }, CancellationToken.None);
   }
   finally
   {
       Interlocked.Decrement(ref _activeTasks);
       linkedCts.Dispose();
   }
   ```
4. The contextPrompt parameter should include: agent name, current game state from SessionManager, and the passed `context` string. Format as a system-like preamble: "You are [agent]'s analytical brain. Analyze this game screenshot. Context: {context}"

**IBrainService.SubmitQueryAsync(string userQuery, SharedContextEnvelope context, CancellationToken ct):**
- Similar pattern to SubmitImageAsync but:
  - Uses text-only message (no vision content)
  - Prepends context envelope as system context
  - Result type is `BrainResultType.ToolResult`
  - Uses `_workerModel` instead of `_brainModel` (queries are lighter weight)

**IBrainService.CancelAll():**
- Cancel `_globalCts`
- Create new `_globalCts = new CancellationTokenSource()`

**Dispose:**
- CancelAll()
- Complete the channel writer: `_channel.Writer.TryComplete()`

**Private helper:**
- `static string TruncateForVoice(string? text, int maxLen)` — truncates at sentence boundary if possible, falls back to hard truncate with "..."

**Important implementation notes:**
- The context prompt for SubmitImageAsync should be: "You are a gaming AI analyst. Analyze this game screenshot and provide tactical insights. Current game: {sessionManager.Context.GameType ?? "unknown"}. {context}"
- Use the OpenRouterClient methods — do NOT create HttpClient directly
- All writes to channel use WriteAsync (not TryWrite) to respect backpressure, but with CancellationToken so they don't block forever
- The tool call loop checks `FinishReason == "tool_calls"` — this is OpenAI's convention that OpenRouter follows
