    **4. Add session restart capability to MainViewModel:**
    ```csharp
    public async Task RestartSessionAsync()
    {
        if (ConnectionState != ConnectionState.Connected) return;
        var agent = SelectedAgent;
        var target = SelectedTarget;

        // Disconnect current session
        await _conversationProvider.DisconnectAsync();
        await StopSessionAsync();

        // Brief pause for cleanup
        await Task.Delay(100);

        // Reconnect with same agent + target (provider factory reads new settings)
        if (agent != null && target != null)
        {
            SelectedAgent = agent;
            SelectedTarget = target;
            await ToggleConnectionAsync();
        }
    }
    ```

    Access MainViewModel from SettingsPage via the DI service provider. Use the EXACT pattern already established in MainPage.xaml.cs line 35:
    ```csharp
    var mainVm = Application.Current?.Handler?.MauiContext?.Services.GetService<MainViewModel>();
    ```
    This is the project's established convention for resolving singleton services from code-behind. Do NOT use MessagingCenter or WeakReferenceMessenger.

    After confirmation + save:
    ```csharp
    var mainVm = Application.Current?.Handler?.MauiContext?.Services.GetService<MainViewModel>();
    if (mainVm != null)
        _ = mainVm.RestartSessionAsync();
    ```
